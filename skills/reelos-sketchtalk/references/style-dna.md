# 风格 DNA

## 一句话

白底、黑字、红色命题、强排版、大留白、极简隐喻图、轻哲思、内容优先的短视频观点页。

像一页正在说话的思想卡片：先用文字压住注意力，再用底部一幅极简图把情绪落地。

## 必须

- 9:16 竖屏画幅，适合短视频封面、分镜页、小红书/抖音知识卡片。
- 纯白背景，可有极轻灰阶层次，但不要纸纹、渐变、噪点、阴影。
- 文字是主体，插画是情绪承接，不要反过来。
- 顶部署名从用户输入推导，例如作者、账号、栏目名、系列名；没有提供时可以留空或使用中性占位，不要默认塞入 ReelOS。
- 红色命题句放在上部，像一句问题或世界观判断。
- 黑色主观点放在视觉中心，粗、大、短，最多 2-3 行。
- 英文副标使用细字距大写，从主题或栏目名转写而来；只有用户明确要求 ReelOS 系列时才使用 `REEL OS SKETCH TALK`。
- 底部只放一个极简隐喻场景，占画面约 20%-30%。
- 普通人物剪影或半抽象人物可以出现，但不能成为吉祥物或固定 IP。

## 颜色

- 白色：主背景。
- 黑色：主观点、人物剪影、主要线稿。
- 克制暖红：命题句、太阳/月亮/果实/重点点状元素，使用要克制。
- 软灰：顶部信息、次级文字、极轻分隔或底部 UI 安全区外的弱化层。

推荐色感：

- Black: `#111111`
- Warm Red: `#E84A4A` 或接近的暖红
- Soft Gray: `#B9B9B9`
- Background: `#FFFFFF`

## 文字气质

- 中文主观点：厚重黑体、现代、强对比，像视频封面标题。
- 红色命题句：中等粗细，长句可以横向铺开，形成开场语气。
- 英文副标：细、窄、拉开字距，承担节奏，不承担信息量。
- 右下角可用小字：`个人观察` / `仅供参考`，但不是每张都必须有。

## 视频模板层级

如果 SketchTalk 样式被用于口播视频，不要把所有文字都做成封面强度。

- 封面：主观点最大，2-3 行，强黑字，负责吸引点击。
- 视频内 beat 字幕：跟随 TTS 原文短句，比封面标题小 10%-20%，左右留更大安全边距。
- 底部辅助字幕：更小、更淡，只做同步提示；不要形成第二个标题。
- 底部隐喻图：保持同一物件系统，但要随口播短句轻微变化，让画面有生命力。

已验证方向：

- 白底、黑字、暖红命题句、灰色英文副标。
- 顶部使用集数 + 系列名 + 小号英文栏目。
- 中心标题例：`感知 / 是一场 / 可控幻觉`。
- 底部隐喻例：门、路径、红点、小人物，用来表达“世界版本被生成”。

## 绝对不要

- 不要复刻参考图里的账号名、圆形“深”章、底部平台 UI 或原视频文案。
- 不要使用外部品牌、水印、视频播放器控件，也不要强行加入 ReelOS 品牌。
- 不要做成 PPT、课程封面、营销海报或复杂信息图。
- 不要堆节点、箭头、流程框。
- 不要可爱卡通、商业插画、写实人物、复杂场景。
- 不要让红色覆盖大面积，除非是极简太阳/月亮这类单一视觉锚点。
- 不要把页面塞满；留白就是风格的一部分。
- 不要把视频内每一句 beat 字幕都做成封面级巨字。
- 不要让底部辅助字幕、英文副标和主字幕同时抢注意力。

## 审美方向

冷静、有力量、克制、轻哲学、适合口播停顿。

不要热闹、鸡血、可爱、复杂、炫技。
