# 竖屏版式模板

## 基础画幅

- 默认 9:16。
- 推荐生成尺寸语义：vertical poster, short-video cover, 1080x1920 composition。
- 画面上下都要留安全区，避免平台控件遮挡底部插画和右下小字。

## 区域比例

从上到下：

1. 顶部署名区：0%-10%
2. 红色命题句：12%-24%
3. 主观点区：30%-45%
4. 英文副标区：47%-54%
5. 留白呼吸区：55%-66%
6. 底部隐喻图：66%-88%
7. 右下角小字或空白：88%-96%

比例可以按文字长短微调，但不要让插画上移压迫主观点。

## 已验证封面基准

适合系列口播封面，尤其是心理学、哲学、心法、认知表达类内容。

```text
[左上：集数]              [顶部居中：系列名]              [右上：小号英文栏目名]


[红色命题句：你以为______，其实______]


[粗黑主观点：概念 / 判断 / 结果]
[2-3 行，封面级最大字体]

[细字距英文副标]


[底部极简隐喻图：门 / 洞 / 路径 / 镜 / 红点 / 小人物]


[底部关键词脉络：事实 | 脑补 | 旧伤 | 观心]
```

封面主观点可以比视频内 beat 字幕大。视频内主字幕必须降一级，避免每一帧都像封面。

## 已验证视频内页基准

适合 SketchTalk 口播视频正文帧，尤其是《空见心力》这类心理学、哲学、心法、认知表达系列。

```text
[左上：当前页序号 + 细进度线]      [顶部居中：系列名 / 章节名]


[红色场景提示：开场提问：先看见你如何向自己发问]
[或：遇到任何事，按顺序问自己三个问题。]


[粗黑主 beat：TTS 当前短句]
[2 行以内，字号低于封面主观点]

[底部辅助：方法名 / 02-05]
[英文副标：ASK A BETTER QUESTION]


[底部隐喻图：随 beat 改变的极简图形]
[例如三节点：事实 → 原因 → 行动]


[底部固定关键词脉络：问题 | 方向 | 行动 | 杠杆 | 复盘 | 选择]
```

视频内页的信息职责：

- 红色提示：只负责定场、提出问题或提示方法，不直接重复主 beat。
- 主 beat：只显示当前真实口播短句，是页面主阅读。
- 底部辅助：只显示方法名、章节名和 beat 进度，例如 `事实、原因、行动，别跳步 / 01-13`。
- 英文副标：只负责系列节奏，不解释正文。
- 底部隐喻图：必须跟当前段落语义一致，并随 beat 产生状态变化。

如果同一句话同时出现在红色提示、主 beat、底部辅助三处，必须重排；优先保留中间主 beat，顶部改成场景提示，底部改成方法提示或进度。

## 结构模板

```text
[可选左上小标]                  [顶部居中：用户输入的作者 / 账号 / 栏目名 / 系列名]


[红色命题句：所有人都在解决一个问题：______]


[黑色主观点：2 行以内，强判断]
[可以换行，但不要超过 3 行]


[由用户内容推导的英文副标]



[极简隐喻图：人物 / 山 / 路 / 门 / 太阳 / 洞 / 线团]


                                           [个人观察]
                                           [仅供参考]
```

## 文案层级

### 顶部署名

优先使用用户输入里的内容，不要默认使用 ReelOS。

可选：

- 用户给出的作者名、账号名、栏目名或系列名
- 用户给出的文章标题缩写
- 无署名需求时留空
- 只有用户明确要求 ReelOS 系列时，才使用 `漫说 ReelOS`、`@ReelOS` 或 `ReelOS SketchTalk`

### 红色命题句

功能：提出大问题或铺设语境。

句式：

- `所有创作者都在解决一个问题：如何______`
- `真正困难的不是______，而是______`
- `你以为你缺的是______，其实你缺的是______`
- `当内容开始变多，系统才会暴露真正的问题`

### 黑色主观点

功能：封面级核心判断。

要求：

- 10-28 个中文字最稳。
- 最多 2-3 行。
- 宁可短，不要完整解释。
- 不要标点太多。

### 英文副标

默认从用户输入推导：

- 文章标题可转写为英文副标
- 核心主题可转写为短英文词组
- 无明确英文需求时可使用拼音或留空

可根据系列变体使用：

- `ATTENTION OPERATING SYSTEM`
- `FATE AND ATTENTION`
- `MAN SHUO REELOS`
- `REEL OS THINKING`
- `BUILD IN PUBLIC`

英文只负责节奏和系列感，不抢中文观点。

## 版式变体

### A. 封面型

- 主观点最大。
- 底部隐喻图更简。
- 适合视频封面和系列首图。
- 顶部信息保持克制：集数、系列名、小号英文栏目即可，不加平台元素。
- 底部关键词 3-5 个即可，当前主题词用暖红点亮。

### B. 分镜型

- 主观点略小。
- 红色命题句可以连续承接上一页。
- 底部隐喻图表达动作变化。
- 如果做成视频模板，分镜型主字幕必须跟 TTS 原文短句同步；视觉摘要词只能做页面标题或 footer 关键词。
- 底部隐喻图每句可有轻微横移、路径显影、红点激活或 2%-4% 微缩放。
- 如果口播稿改动，必须同步更新分镜主字幕、cue、底部辅助和隐喻图；不要让旧隐喻继续服务新段落。

### C. 纯观点型

- 插画极少，只保留红点、路径、人物剪影。
- 适合一句非常强的观点。

### D. 情绪落点型

- 插画稍微更完整，例如山谷、桥、门、月亮。
- 适合片尾、总结、转折页。
