# Professional Reading Methodology

Use this protocol when turning a book into reusable knowledge.

## Mission

Extract three things from the book:

- Core principles: underlying rules that survive practice.
- Opposing views: especially counterintuitive claims and where common sense fails.
- Executable methodology: what to do, in what order, and why that order matters.

For the reading process itself, combine this protocol with `onion-reading-system.md`.

## Thinking Priorities

Before writing, internalize this order:

1. Problem-driven: ask what problem the user reads this book to solve.
2. Structure-first: show the skeleton before details.
3. Concept-first: define key terms before stories and cases.
4. Card-solidified: every knowledge point should become a reusable card.
5. Transfer-oriented: extract principles, boundaries, and adjacent domains.
6. Output-driven: every section should answer what the reader can do after reading.

## Startup Protocol

When a new book task starts, parse or ask for:

```text
【新书任务启动】

书名：《___》
作者：___
我的读前三问：
  1. 我为何读这本书？
  2. 我要解决什么具体问题？
  3. 读完我打算做什么？

阅读模式：□碎弹 □爆破 □核弹
```

If the user did not provide the three questions, infer a default from the book topic and mark it as an assumption.

## Reading Modes

- 碎弹模式: 15 minutes. Fragment/thematic scan; output core thesis, structure skeleton, counterintuitive list, and 3 action cards.
- 爆破模式: 90 minutes. Book-based thematic reading; output the full six-layer flow and a 33-card framework.
- 核弹模式: cross-book thematic reading. Compare 3 books on the same issue and output a principle map.

Default to 爆破模式 for serious PDF book tasks. Use 碎弹模式 for quick poster-only tasks. Use 核弹模式 only when multiple books or a theme corpus exists.

## Default Output Sections

Every serious reading deliverable should include:

- 核心主张: 1-3 sentences that capture the book's nucleus.
- 结构骨架: a directory-level framework map.
- 反常识洞见: at least 3 counterintuitive claims.
- 方法论卡片: an ordered action sequence.
- 知识卡片: cards with definition, boundary/counterexample, and application scenario.
- 九宫格: 书名 / 问题 / 阿哈 / 概念 / 核心 / 案例 / 书单延伸 / 感悟 / 接下来做.
- 读后行动闭环: 3 questions answered, action target, gap, MVP, iteration signal.

## Card Quotas For 爆破模式

Produce a 33-card framework:

- 概念卡 x10
- 框架卡 x6
- 问题卡 x5
- 案例卡 x5
- 行动卡 x5
- 总结构卡 x2

Each card should contain at least two of:

- 定义
- 反例或边界
- 应用场景

## HTML Delivery

Final delivery is HTML. Use:

- One-page poster HTML for quick capture.
- Full reading-notes HTML for detailed reading.
- Markdown only as an intermediate source file.
