# Reading Output Schema

Use this schema unless the user requests a different format.

Final user-facing delivery should be a standalone HTML file. Markdown is an intermediate notes format.

Use `professional-reading-methodology.md` as the controlling schema for serious book tasks. The sections below are fallback primitives.
Use `onion-reading-system.md` as the controlling schema for the reading workflow itself.
Use `zhang-kai-fast-reading.md` as the fast-reading gate: evaluation, speed pass, rest, deep read, forced output.
Use `wang-zhuan-learning-system.md` when a reading task should become writing, a knowledge system, workplace sharing, or practice feedback.
Use `knowledge-asset-workflow.md` when the final deliverable should become reusable knowledge assets.

## Quality gates

- Evaluation must produce one of four decisions: 继续读, 只速读, 跳读, 暂不读.
- Unknown-book fallback must stay neutral. It may produce a reading scaffold, but must not invent a final thesis or borrow concepts from a known-book adapter.
- Full HTML must be standalone and offline-readable.
- Sidebar anchors must be generated from real Markdown headings.
- Final output must not contain placeholders such as TODO, 待补, 未检测到明确, or 定义 / 边界 / 应用场景.
- 爆破模式 must either provide 33 meaningful cards or explicitly keep the 33-card framework in a deep-reading appendix for later confirmation.
- Every methodology card must include at least two of: definition, boundary/counterexample, application scenario.
- The action loop must include an action goal, MVP, success/failure signal, and iteration rule.
- Single-concept repetitive books must include one-concept diagnosis, a concept model, question chain, keyword map, gold-sentence/paraphrase bank, and Feynman card.
- Direct gold-sentence quotes must be short and sparse; prefer paraphrase when copyright or source confidence is uncertain.
- Learning-method outputs must include a write-do feedback loop: thought, card, explanation/article, practice, feedback, revised view.
- Knowledge-asset outputs must include reading goal, book map, reading priority, synthesis, and at least one reusable asset.

## Professional default

- 核心主张: 1-3 sentences.
- SVG 思维模型图: model name, input, mechanism, output, intervention point, usage formula.
- 阅读目标: quick overview, deep study, writing/content, product/startup, management, investment/business research, or personal action.
- 全书地图: one-sentence book, core problem, target reader, structure, chapter relation, reading priority.
- 单概念判断: whether the book is mostly about one thing, one concept, or one model.
- 结构骨架: directory-level framework map.
- 提问链: pre-reading, during-reading, and post-reading questions that guide extraction.
- 关键词地图: core terms, support terms, action terms, boundary terms.
- 金句/可转述句: short original gold sentences when safe, plus agent-owned paraphrases and usage scenes.
- 费曼讲解卡: explain to a child, a colleague, and the user's next action.
- 三类知识卡: 观点卡, 方法卡, 案例卡.
- 写作外化: one short explanation, article outline, or teaching script that makes thinking visible.
- 体系化输出: system map, series outline, or theme research report structure.
- 反常识洞见: at least 3 counterintuitive claims.
- 章节精读模板: one-sentence summary, main idea, concept, case/evidence, reasoning chain, weak assumption, user insight, action.
- 全书复盘: core framework, 10 key ideas, 5 actions, 3 questions, most valuable chapters, skim/skip chapters.
- 知识资产转换: one-page card, executive summary, article/social post, team sharing/PPT outline, product/business/management implications, personal action plan.
- 方法论卡片: ordered action sequence.
- 知识卡片: definition, boundary/counterexample, application scenario.
- 九宫格提取: 书名, 问题, 阿哈, 概念, 核心, 案例, 书单延伸, 感悟, 接下来做.
- 行动闭环: 3 questions answered, next action, gap, MVP, iteration plan.
- 反馈闭环: expected result, feedback cycle, milestone, review record.
- 已知领域连接: similar concept, transferable principle, non-transfer boundary, user-domain example.
- 跨场景迁移矩阵: trigger, old reaction, new action, failure boundary, validation signal.
- 7 天复盘日志: date/day, reps, resistance, old behavior occurred, next adjustment.
- 教学输出包: 60-second explanation, 3 analogies, 5 questions, 1 exercise, common misunderstandings.

## Depth modes

- 碎弹模式: core thesis, structure skeleton, counterintuitive list, 3 action cards.
- 爆破模式: full six-layer flow and 33-card framework.
- 核弹模式: 3-book comparative principle map.

## Quick skim

- Book identity: title, author if detectable, PDF filename
- Best use case: why this book is worth reading
- Continue / skim / skip recommendation
- 5 core ideas
- 3 questions the book can answer
- Sections to deep-read
- Sections to skim

## Deep reading

- Reading goal
- Book thesis
- Argument map
- Chapter-by-chapter notes
- Key concepts and definitions
- Evidence, examples, and case studies
- Actionable methods
- Weak assumptions or gaps
- Connections to user's context

## Implementation notes

- Action
- Source idea
- When to use it
- Steps
- Risks
- First experiment

## Review cards

Use concise cards:

- Q: What problem does this idea solve?
  A: ...
- Q: When should I apply it?
  A: ...
- Q: What is an example?
  A: ...

## Reading plan

- 30-minute plan: orientation plus top sections
- 2-hour plan: core chapters plus notes
- Full plan: deep-read sequence with outputs after each chapter
