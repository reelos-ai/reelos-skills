# 王专学习方法提炼

## 核心主张

学习不是“多输入”，而是把抽象道理变成具体行动，再用写作和反馈不断修正认知。

可压缩为四步:

1. 知行合一: 思中有做，做中有思。
2. 写作合一: 写作不是记录思考，而是思考本身。
3. 构建体系: 用卡片、图谱、系列文章把散装知识变成整装知识。
4. 实践应用: 用行动、反馈、小胜、复盘验证知识是否真的有用。

## 吸收到读书 skill 的原则

- 提问优先: 少问“凭什么”，多问“为什么”“如何才能”“下一步怎么做”。
- 关键词优先: 读书时先抓核心词、行动词、边界词，而不是平均摘录。
- 金句压缩: 金句不是收藏品，而是压缩模型、方便记忆和表达的工具。
- 写作外化: 大脑会美化自己的思考。没写出来，就不能假设自己想清楚了。
- 卡片集装箱: 每个知识点装成一张小卡，降低整理成本，方便后续组合。
- 体系化输出: 读完不只做摘要，还要画体系、写系列文章或主题研究报告。
- 实践反馈: 学到的东西必须转成行动目标、预期结果、反馈时间点和复盘记录。

## 知行合一

真思考不是空想，而是能连接现实行动。真行动不是盲做，而是在行动中观察、反思、修正。

读书时的应用:

- 看到观点，追问: 它能改变我哪个具体行动？
- 看到方法，追问: 我可以在哪个真实场景试一次？
- 看到案例，追问: 它背后的规律是什么，边界在哪里？

## 写作合一

写作让抽象思考变得具体。写不清，通常说明还没想清。

读书时的应用:

- 想了要写: 把模糊理解写成一句话。
- 写了要做: 把一句话转成一个动作。
- 做时要写: 行动中记录观察和反馈。
- 做后再写: 用复盘修正观点和表达。

## 卡片体系

卡片像集装箱。不要急着写大文章，先把灵感、观点、方法、案例装成小卡。

三类基础卡:

- 观点卡: 怎么想。记录判断、原则、反常识、核心主张。
- 方法卡: 怎么做。记录步骤、顺序、条件、风险。
- 案例卡: 直观。记录故事、场景、对比、反馈。

好卡片至少包含两个要素:

- 一句话定义或判断。
- 使用边界或反例。
- 应用场景或下一步动作。

## 主题式学习

主题式学习不是找唯一答案，而是拼拼图。

执行方式:

1. 围绕一个具体问题收集不同观点。
2. 故意读一些可能反对自己的观点。
3. 找概念之间的关系，而不是堆材料。
4. 输出体系图和系列文章。
5. 用实践反馈继续修正体系。

## 高价值观点的七个入口

- 换角度: 从不同角色、立场、时间尺度看问题。
- 挖本质: 从现象追到规律。
- 正反看: 同时问“为什么成立”和“什么时候不成立”。
- 调范围: zoom in 看细节，zoom out 看系统。
- 系统看: 关注元素之间如何互动。
- 实用看: 观点能不能在现实中用。
- 决策看: 观点能不能帮助做更好选择。

## 清晰表达

表达的目标不是显得聪明，而是让别人真的理解。

有效手段:

- 类比和费曼: 像智者一样思考，像普通人一样说话。
- 举例: 用具体生活场景承载抽象道理。
- 图示: 用流程图、体系图、对比图降低理解成本。
- 讲故事: 用情境和冲突帮助记忆。
- 连接旧知识: 用读者已有知识解释新观点。
- 讲清为什么: 不只说结论，也说动机和原因。

书面表达三条硬规则:

- 删除多余内容。
- 长句拆短句。
- 分段控制阅读节奏。

## 实践反馈

避免最小阻力原则: 人会倾向于做简单、无关紧要、反馈模糊的事情。

读书后的实践设计:

1. 写下关键行动和预期结果。
2. 设置反馈周期。
3. 用小胜保持推进。
4. 用复盘记录真实效果。
5. 把有效经验写成最佳实践，便于共享。

## 交付物标准

当读书目标是学习方法、管理、成长或知识体系建设时，最终 HTML 应优先包含:

- 一页核心图: 四步闭环或核心模型。
- 提问链: 为什么、如何、怎样才能、下一步。
- 关键词地图: 核心词、行动词、边界词。
- 金句与可转述句: 用于记忆、表达和行动提醒。
- 三类知识卡: 观点卡、方法卡、案例卡。
- 体系图: 概念关系、流程关系或实践闭环。
- 主题式研究报告模板: 体系图、系列文章、观点优化。
- 实践反馈表: 目标、预期、行动、反馈周期、复盘。
