# 中文读书笔记润色规则

在脚本生成 `reading-notes.zh-CN.md` 后，用本规则把骨架改成可读的中文笔记。

## 自动化边界

`generate_reading_notes.py --lang zh-CN` 可以做两种事:

- 检测到已知书籍结构或已沉淀适配器时，直接生成更完整的中文精读版。
- 对未知书籍，生成中文骨架和提取线索，再由 agent 按本规则精读润色。

不要让脚本假装能理解所有书。确定性脚本负责结构、目录、提取和已知模式；真正的跨书综合由 agent 完成。

## 处理顺序

1. 保留书名、作者、来源、页数、章节地图。
2. 将“原文提取章节要点”压缩为中文“章节要点”，不要逐句翻译。
3. 扩写“中文深读摘要”，覆盖主张、机制、方法、边界和可执行应用。
4. 保留 ARISE 练习表，并把原文摘录线索改写成中文操作说明。
5. 加入 5-10 张复习卡片，问题要能帮助用户复述和使用本书。

## 中文摘要结构

- 这本书在解决什么问题
- 作者的核心机制是什么
- 这套方法怎么操作
- 为什么它不同于意志力、普通肯定句或鸡汤
- 适用边界和不适用场景
- 用户今天可以做的第一个实验

## 质量标准

- 用中文自然表达，不保留英文长句结构。
- 把英文术语先保留原词，再给中文解释，例如 `frequency counter`（熟悉性计数器）。
- 区分“作者主张”和“我的应用建议”。
- 不要生成长篇逐章流水账；优先保留能指导行动的内容。
- 不要大量引用原文；需要原文时只用短摘录。
