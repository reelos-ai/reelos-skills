# 单概念反复展开型书处理法

## 识别信号

一本书如果出现以下特征，按“单概念反复展开型”处理，而不是逐章平铺:

- 目录看似很多章节，但每章都在围绕同一个核心概念换场景解释。
- 高频关键词高度集中，核心概念不超过 3 个。
- 后半部分多为案例、应用、类比、重复劝说，而不是新增理论。
- 作者经常把同一句主张换成不同表达、不同故事、不同领域。

## 阅读策略

1. 先找“一件事”: 这本书到底反复讲哪一个问题？
2. 再找“一个概念”: 作者用哪个概念解释这个问题？
3. 再找“一个模型”: 概念如何运转，输入、过程、输出分别是什么？
4. 只深读 2-3 个代表场景: 选最能证明概念、最能迁移到用户问题的章节。
5. 跳过重复低增量章节: 如果只是换例子重复核心主张，做一句话记录即可。
6. 最后输出一个动作: 用一个最小实验验证这个概念是否能改变行为。

## 必备输出

- 单概念判断: 这本书是否主要讲一件事？如果是，写出这件事。
- 核心概念卡: 概念名、定义、反例/边界、应用场景。
- 模型图: 刺激/输入 -> 机制 -> 结果 -> 可干预点。
- 提问链: 读前问、读中问、读后问，问题必须能驱动取舍。
- 关键词地图: 核心词、支撑词、行动词、边界词。
- 金句/可转述句: 优先用自己的话压缩作者主张；直接引用必须短、少、标页码。
- 费曼讲解卡: 面向小孩、同事、自己行动三个版本。

## 金句规则

金句不是为了收藏，而是为了压缩模型。

- 原文短句: 只收能代表作者核心主张的短句。
- 自己转述: 把原句改写成更可讲、更可用的话。
- 使用场景: 标注这句话在什么场合能帮助理解或行动。
- 风险边界: 标注这句话什么时候会被误用。

## 读法取舍

- 概念定义、模型结构、反常识论证: 精读。
- 最能迁移到用户问题的 2-3 个案例: 精读。
- 同质化案例、重复劝说、扩展故事: 速读或跳读。
- 参考文献、边缘例子、作者自我重复: 只在追证据时读。
