# 视频终审验收与返修闭环

## 目标

终审验收不只检查“能不能播放”，还要判断视频是否能在前 3 秒抓住注意力、是否持续清晰、是否有品牌记忆点，以及特效是否真正帮助理解。

固定验收链路：

```text
静帧审查
→ 低清全片预览
→ 四种观看模式
→ 100 分评分
→ 硬门槛检查
→ 定向返修
→ 终版技术验收
→ 交付
```

## 一、验收产物

每条完整视频至少保留：

- 首帧、结尾帧、每个 scene 边界帧。
- 每幕 15% / 50% / 85% 中间帧。
- 关键帧 contact sheet。
- 低清全片预览，例如 960×540、15fps 或代理视频。
- 最终 MP4。
- `FinalAcceptanceReport.md` 或同等项目记录。

产品宣传、数据科普和 `slides-video` 必须同时保留 `VFXEnhancementPlan` 与 `FinalAcceptanceReport`。

## 二、四种观看模式

### 1. 静音观看

检查画面是否独立表达趋势、因果、流程、证据和结论。静音后完全无法判断内容，说明视觉只是装饰。

### 2. 只听音频

检查口播是否自然、有没有截断、突兀停顿、语速异常和响度跳变。画面不能掩盖声音问题。

### 3. 正常速度完整观看

不暂停、不拖动，记录注意力下降、信息过载、重复构图和无意义等待的位置。重点检查前 3 秒、每次章节切换和 CTA。

### 4. 暂停逐帧观看

检查文字溢出、重叠、图表标签、鼠标位置、遮罩边缘、转场中间态和产品界面真实性。

## 三、100 分终审评分

| 维度 | 权重 | 合格标准 |
| --- | ---: | --- |
| 钩子与留存 | 20 | 第一帧完整，前 3 秒建立问题、反差或产品信号，没有空白等待 |
| 语义清晰 | 20 | 动效能直接解释当前关系，静音也能识别主要内容 |
| 音画同步 | 15 | 主动作、卡拉 OK、数字锁定与真实 TTS cue 对齐 |
| 层级与可读性 | 15 | 标题、字幕、图表和界面焦点明确，没有争抢、遮挡和溢出 |
| 镜头连续性 | 10 | 转场承接位置、方向、对象或语义，不是模板随机切换 |
| 酷感与品牌记忆 | 10 | 至少一个有语义的英雄时刻，整体风格符合品牌而非通用科技模板 |
| 多样性与节奏 | 5 | 相邻镜头不重复构图，强弱有起伏，留白和运动分配合理 |
| 技术稳定 | 5 | 按帧确定、无抖动、无异常黑帧、渲染和解码通过 |

判定：

- `85–100`：可以交付。
- `75–84`：必须完成一轮定向返修后重新评分。
- `<75`：不能交付，回到 VFXEnhancementPlan 或分镜层重做。
- 总分达到 85 但任一核心维度低于其权重的 70%，仍不通过。

核心维度为：钩子与留存、语义清晰、音画同步、层级与可读性。

## 四、硬门槛

出现以下任一问题时直接阻止交付，不以总分抵消：

- 第一帧空白、半入场或品牌和标题不可识别。
- 产品事实、数字、界面或数据来源不真实且未标记为示意。
- 标题、字幕或图表标签溢出、遮挡、无法阅读。
- 卡拉 OK 词级高亮明显错词，或主视觉动作长期偏离 TTS cue。
- 非剧情需要的连续黑帧超过约 0.2 秒。
- 大号文字持续抖动、布局跳变或镜头造成明显眩晕。
- 连续三个 scene 使用相同构图和相同主动作。
- 使用不可按帧复现的随机运动、CSS animation 或浏览器计时器。
- 完整渲染无法解码，缺少音频/视频流，或时长明显异常。

同步参考：

- 卡拉 OK 词级高亮目标误差控制在约 80–120ms；素材条件不足时不超过 160ms。
- 主视觉动作目标落在对应重音或句意完成点附近，明显偏差超过约 250ms 时应校准 cue。
- 不通过简单拖动整条时间轴掩盖局部问题，应修改具体 beat offset。

## 五、返修闭环

最多进行两轮创意返修，之后必须降级为稳定方案，不能无限叠加复杂度。

### 第一轮：定向修正

保留结构，只修具体问题：

- 钩子弱：提前产品或核心反差，强化第一可见动作。
- 语义弱：换成关系更直接的图表、路径或状态变化。
- 不够酷：强化一个英雄时刻，不给每幕同时加特效。
- 重复：改变视觉重心、构图或镜头方向，不只换 easing。
- 同步偏差：调整具体 cue 和词级时间，不估算整段时长。
- 过载：删除辅助动作，降低氛围层，拆分信息。

### 第二轮：结构性修正

重新分配 scene、强度曲线或转场连续性：

- 合并重复 scene。
- 拆开高密度 scene。
- 重排英雄时刻。
- 将重特效替换为预烘焙素材或轻量 SVG。
- 把不稳定的实时 blur、粒子或大图层降级。

### 最终降级

两轮后仍不稳定时，回退到稳定排版、真实证据、局部高亮和简单路径。清晰且可靠的低复杂度方案优于不稳定的炫技方案。

## 六、终版技术验收

至少执行：

```bash
npm run typecheck
npx remotion render src/index.ts <CompositionId> out/<slug>.mp4
ffprobe -v error -show_streams -show_format -of json out/<slug>.mp4
ffmpeg -v error -i out/<slug>.mp4 -f null -
```

建议增加：

```bash
ffmpeg -hide_banner -i out/<slug>.mp4 -vf "blackdetect=d=0.20:pix_th=0.02" -an -f null -
ffmpeg -hide_banner -i out/<slug>.mp4 -af loudnorm=I=-16:TP=-1.5:LRA=11:print_format=json -f null -
```

必须确认：

- 分辨率、帧率、时长符合目标。
- H.264/AAC 或用户指定格式正确。
- 音频与视频流都存在。
- 全片解码无错误。
- 无非预期连续黑帧。
- 网络视频建议响度接近 -16 LUFS，除非平台或用户另有要求。
- 第一帧可以直接作为封面，CTA 有足够稳定停留。

最终标准：既要让观众看懂，也要让观众愿意继续看；既要有记忆点，也必须可稳定交付。
