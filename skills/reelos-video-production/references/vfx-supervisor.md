# 特效师与视觉强化

## 角色定位

特效师（VFX Supervisor）位于自动动效导演和 Remotion 实现之间。动效导演负责“为什么动、什么时候动”，自动动效导演负责“哪个候选最能解释语义”，特效师负责“怎样让选中动作更有冲击力、连续性和品牌记忆，同时不损害理解与稳定性”。

固定分工：

| 角色 | 主要问题 | 必须产出 |
| --- | --- | --- |
| 视觉设计师 | 静态画面是否有正确的品牌气质与信息层级 | Visual Design Card |
| 动效导演 | 全片用什么运动骨架讲清内容 | Motion Brief |
| 自动动效导演 | 当前 beat 最适合哪个语义动作 | MotionDecisionPlan |
| 特效师 | 如何强化冲击力、镜头连续性与英雄时刻 | VFXEnhancementPlan |
| 终审验收 | 成片是否真正清晰、吸引、稳定并可交付 | FinalAcceptanceReport |

特效师不能推翻已经确认的口播、产品事实、视觉设计卡或 Motion Brief。若视觉强化与内容理解冲突，内容理解优先。

## 固定执行顺序

```text
Motion Brief
→ MotionDecisionPlan
→ VFXEnhancementPlan
→ Remotion 实现
→ 低清预览
→ 终审验收
→ 定向返修
→ 终版渲染
```

以下任务必须启用完整特效师环节：

- 产品宣传、品牌短片、发布视频、`slides-video`。
- 数据可视化、投资、AI 系统、工程机制或复杂科普。
- 用户要求“更酷”“更有冲击力”“多一些特效”“像发布会”。
- 时长超过 60 秒且存在 4 个以上章节。

普通低动态口播可以使用简版：只定义 1 个英雄时刻、强度曲线和转场连续性。

## VFXEnhancementPlan 契约

在写组件前，为每个 scene 建立可审计计划：

```ts
type VFXEnhancementPlan = {
  sceneId: string;
  narrativeRole: 'hook' | 'explain' | 'evidence' | 'turn' | 'climax' | 'cta';
  audioCue: string;
  focus: 'caption' | 'chart' | 'path' | 'product' | 'brand';
  primaryEffect: string;
  supportEffects: string[];
  heroMoment?: string;
  intensity: 0 | 1 | 2 | 3 | 4 | 5;
  continuityIn: string;
  continuityOut: string;
  durationFrames: number;
  performanceTier: 'light' | 'medium' | 'heavy';
  fallback: string;
};
```

规则：

- 每幕最多 1 个主特效和 2 个低强度辅助特效。
- `heroMoment` 不是每幕都要有；每个运动章节最多 1 个，全片至少 1 个真正可记忆的瞬间。
- `audioCue` 必须来自最终 TTS timing。
- `continuityIn` 和 `continuityOut` 必须解释镜头如何承接，不写“随机转场”。
- `heavy` 场景必须提供稳定降级方案。
- 特效不能替代真实产品证据，也不能遮住字幕、图表或关键界面。

## 特效库与适用语义

### 文字类

| 特效 | 适用 | 约束 |
| --- | --- | --- |
| 逐字卡拉 OK | 强口播、金句、短字幕 | 必须有词级时间戳或强制对齐；不用于长段全文字幕 |
| Sheen 扫光 | 产品名、品牌名、关键结论揭示 | 同一标题通常只扫一次，不持续循环 |
| 翻牌/里程表数字 | 价格、比例、分数、排名 | 数字必须有真实来源或标记为示意 |
| Kinetic Typography | 观点转折、关键词让位、节奏型短句 | 大字落位后保持稳定，不持续弹跳或抖动 |

### 数据可视化类

| 特效 | 适用 | 约束 |
| --- | --- | --- |
| K 线/折线绘制 | 趋势、行情、增长、成本变化 | 真实数据优先；示意图必须明确方向含义 |
| 雷达扫描 | 事件发现、机会识别、风险侦测 | 扫到节点才点亮，不做无目标旋转 |
| 仪表盘指针 | 势能、风险、信心、完成度 | 读数必须在口播重音附近锁定 |
| 热力矩阵 | 相关性、强弱分布、优先级 | 逐格激活，避免全部同时闪烁 |
| 流向粒子 | 事件到资产、输入到输出、因果传导 | 粒子沿确定路径运行，不能成为背景噪声 |

### 界面叙事类

| 特效 | 适用 | 约束 |
| --- | --- | --- |
| 幽灵鼠标 | 展示真实操作步骤 | 点击必须对应真实交互，不伪造功能 |
| 爆炸拆解图 | UI 层级、证据层、系统架构 | 3–4 层优先，保证每层标签可读 |
| 聚光灯暗场 | 单点讲解、证据聚焦 | 亮区跟随讲解，不能遮住必要上下文 |
| 分屏擦除 | Before/After、旧版/新版 | 擦除线表达比较，不用于无差异画面 |
| 截图拼装 | 产品界面出现、模块汇聚 | 碎片数量受控，最终界面需稳定停留 |

### 镜头与转场类

| 特效 | 适用 | 约束 |
| --- | --- | --- |
| Zoom-through | 从概览深入某个模块 | 下一幕必须是被推进区域的语义延续 |
| Whip Pan | 快节奏章节切换、时间状态转换 | 只在低文字密度处使用，模糊持续不超过约 12 帧 |
| 单帧闪白 | 强信号、断点、结果冲击 | 全片最多 1–2 次，避免连续闪烁 |
| Blur-through | 相关视图、状态转换 | 实时 blur 要限制帧数并准备无 blur 降级 |

### 氛围类

胶片颗粒、透视网格、呼吸辉光、扫描线、地图点位脉冲只允许作为底层氛围。它们不能成为主动作，不能持续改变布局，也不能降低文字和产品界面的对比度。

## 强度曲线与效果预算

用 `0–5` 表示每幕强度：

- `0`：静态停留或必要留白。
- `1`：轻微状态反馈。
- `2`：说明型运动，阅读优先。
- `3`：明确但克制的主动作。
- `4`：章节转折或强证据揭示。
- `5`：英雄时刻，只在最重要瞬间短暂出现。

推荐曲线：

```text
首屏钩子 4
→ 解释段 2–3
→ 关键转折 4
→ 英雄时刻 5
→ 证据段 2–3
→ CTA 3–4
```

预算规则：

- 不允许连续两个 scene 都是强度 5。
- 高密度图表 scene 的镜头运动不超过强度 2。
- 同一主特效连续使用时，第二次必须改变信息任务，不能只换颜色。
- 每 2–4 秒可以有一次有意义的状态变化，但不要求所有元素一直运动。
- 全片变速不超过约 3%；优先调整画面而不是扭曲人声。
- 纯炫技且无法说明语义的特效直接删除。

## 连续性设计

每次转场先判断上下文关系：

| 上下文关系 | 推荐连续性 |
| --- | --- |
| 延续同一对象 | Match move、平移承接、属性变化 |
| 从整体进入局部 | Zoom-through、聚光灯收束 |
| 从旧状态到新状态 | 分屏擦除、形态置换 |
| 从多个候选到结论 | 收束、过滤、对焦 |
| 从机制回到品牌 | 拉远、减速、品牌锁定 |
| 强转折或信号出现 | 短促 Whip Pan 或单帧闪白 |

相邻镜头不能只靠黑场或无意义淡入淡出连接。转场必须继承至少一个元素：位置、方向、颜色、形状、对象或语义。

## 特效师预审

实现前必须回答：

1. 前 3 秒的视觉钩子是什么？第一帧是否已经有完整内容？
2. 全片最值得记住的英雄时刻是什么？
3. 哪些 scene 需要动，哪些 scene 应该停下来让观众读懂？
4. 相邻两幕如何承接？是否连续重复相同构图？
5. 哪些效果属于 `heavy`，如何降级？
6. 静音看时，是否仍能理解当前关系？

最终原则：特效师不是让所有东西都动，而是让最重要的东西在正确时刻，以最有记忆点的方式发生。
