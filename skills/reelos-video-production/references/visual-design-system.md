# 视频视觉设计系统

## 设计来源

本规则吸收 guizang-ppt-skill 的两个可复用思想：先选视觉基调，再从受控主题色板中选一套；一条作品只用一套主题，不临场混搭。也吸收 baoyu-design 的流程思想：真实设计上下文优先、设计系统绑定、交付物自包含、预览验证闭环。还参考 html-video 的模板库，只学习其配色、字体、版式气质和视觉纪律，不整合它的播放器、HTML 渲染引擎、模板工程和交互流程。这里已改写为 ReelOS 口播视频场景，不复制 PPT 模板、HTML 原型和页面结构。

## 视觉设计师职责

视觉设计师环节不是“美化一下”，而是在写 Remotion 组件前完成视觉判断。职责包括：

1. 读内容：判断这条视频是信号、教学、投研、备忘录、强情绪还是品牌观点。
2. 看上下文：优先检查项目已有成功视频、用户指出喜欢或不喜欢的截图、已有组件、配色、字号和动效。
3. 定约束：把已有成功风格、品牌色板、用户选择的风格和平台画幅变成明确约束。
4. 出设计卡：用一张视觉设计卡描述这条视频的色板、版式、明暗节奏、画面主语和禁用方向。
5. 做审稿：导出关键帧和动效中间态，检查它是不是像一个真实团队做的成片，而不是模板拼装。
6. 反写规则：如果这次用户确认某个风格更好，把可复用的色板、构图或动效沉淀回 skill 参考文件。

## 真实上下文优先

开始新视频前，先做一次轻量上下文扫描。能读到什么就读什么，不要为了形式拖慢生产。

优先级：

1. 用户当前浏览器里的成片或 Studio 画面。
2. 用户刚刚明确说“好/不好”的截图和反馈。
3. 项目里已有同类 Composition。
4. `out/stills/`、`out/*.mp4`、`public/` 里的旧成片和资产。
5. 已沉淀的风格预设和视觉设计卡。

扫描后只提炼可执行结论：

```text
保留：
避免：
这次强化：
绑定约束：
```

规则：

- 旧视频被用户确认“不错”时，它就是视觉合同，不能随意换成另一套审美。
- 用户指出“线框难看”“AI 味重”“不如之前”时，要把问题转译成具体禁用项，例如细线框、假 HUD、紫蓝渐变、模板卡片、过密标签。
- 如果没有上下文，才使用默认推荐：战略作战室风 + 墨水经典。
- 如果引用外部设计系统，只吸收方法和审美原则，不复制对方专有素材、文案和结构。

## 视觉设计卡

每次开始 Remotion 合成前，先产出一张内部视觉设计卡。除非用户明确要求，不必长篇展示给用户，但代码实现必须遵守。

```text
上下文结论：
视觉基调：
目标观众：
内容气质：
推荐风格：
推荐色板：
主题图层：
字体层级：
场景节奏：
画面主语：
动效导演接口：
关键帧检查：
禁用方向：
```

决策顺序：

1. 先扫描真实上下文：旧成片、截图、组件、用户反馈。
2. 再判断内容气质：信号、教学、投研、备忘录、强情绪。
3. 再选视频风格：战略作战室风、黑板推演风、商业备忘录风、投研报告风、街头海报风。
4. 再选一套主题色板。
5. 再判断是否需要主题图层：儒释道、哲学、心理学等抽象主题优先 AI 生成图层；真实场景、产品、新闻再找可授权素材。
6. 最后把可运动的画面主语交给动效导演：扫描、压缩、推进、显影、拼合、验证、锁定等动作必须在 motion brief 里明确用途。

## 动效导演接口

视觉设计师只确定“画面能承载什么”和“风格不能破坏什么”，不直接把每段动画随手写死。科普、系统模型和长视频必须把视觉设计卡交给 `references/motion-director.md` 继续处理。

交接格式：

```text
可运动对象：
  - 背景层：
  - 主图表：
  - 关键词节点：
  - 章节锚点：
  - 结论锁定：
风格约束：
  - 字体：
  - 色板：
  - 明暗：
  - 禁用：
```

判断规则：

- 如果内容是因果、流程、系统、投资、Agent、物理机制，优先由动效导演设计路径追踪、变量激活、反馈闭环。
- 如果内容是哲学/心法/观点短片，动效可以更轻，但仍要有可见主运动，例如显影、锁定、推进、收束。
- 视觉设计卡里的“动效”只写约束和可运动对象；具体运动章节、TTS 绑定和关键帧检查写进 motion brief。

## 主题图层

主题图层是内容与画面之间的“隐性背景语义”。它可以来自 AI 图片模型生成，也可以来自可授权真实素材，但必须服务内容主题，而不是为了好看而堆图。

来源选择：

- 儒释道、哲学、心理学、抽象原则、心法、方法论：优先用 AI 图片模型生成统一风格的背景层、金描线稿和主题隐喻图。
- 真实人物、真实产品、新闻事件、具体地点、案例拆解：优先使用可授权素材、官方图、公共版权或用户提供资产。
- 商业投研、AI 工程、品牌策略：优先用 Remotion 图形系统；只有需要空间氛围或真实质感时再加素材层。

使用原则：

- 先从内容判断主题，再决定搜索关键词和素材类型，不按风格随机贴图。
- 抽象主题优先生成：背景、线稿和纹理要为这条视频定制，而不是找通用库存图。
- 真实素材优先用于：照片、壁画、雕塑、纹理、古籍、建筑、自然景观、数据图形或公共版权文献图。
- 免费素材也必须记录来源、作者、页面 URL、下载日期和许可类型，便于后续追溯。
- Pinterest 只做灵感搜索和关键词扩展，不作为直接素材来源。
- 落地时必须二次设计：裁切局部、降低饱和、加色板统一、模糊或颗粒化、叠加遮罩，避免像素材直接贴上去。
- 一条视频只选择 1 个主图层系统，不能每幕都换一套视觉语言。
- 不要求每一屏长得一样：图层系统保持一致，但每幕可以改变裁切、显隐、比例、位置、动效和信息结构。
- AI 生成主题图层默认是背景替代系统，不叠加在既有重背景、粒子、全屏渐变或素材视频之上；启用后只保留一个主背景系统。
- 落地前先离线优化为 1920x1080 JPG/WebP，把裁切、调色、暗角和遮罩尽量预烘焙，原始 PNG 只作为母版留存。

AI 生成图层模式：

- 背景层：无文字、无 logo、无水印；暗场、纸纹、壁面、山雾、空间、光影，留出左侧或中部文字安全区。
- 金描线稿层：透明/纯底输出更佳；人物、法相、道人、学者、心智隐喻只做轮廓，不画五官，不做主角。
- 主题隐喻层：抽象器物、山水、经卷、星图、坐标、心智结构；只提供场域和隐喻，不解释正文。
- 每条视频通常生成 2-3 张：封面强氛围、中段轻纹理、结尾收束图。
- 默认使用 `public/generated/{slug}-optimized/` 进入 Remotion，避免在合成阶段实时处理大 PNG。
- 默认静态显影或极轻遮罩，不做逐帧 `filter`、大幅 `scale()`、`mixBlendMode` 和长时间全屏漂移。
- 生成提示词详见 `references/ai-image-layer-prompts.md`。

推荐素材来源：

- Pexels：适合照片、自然、建筑、人物氛围、视频背景。
- Unsplash：适合高质量摄影、纹理、空间、自然、宗教建筑、抽象光影。
- Pixabay：适合图片、插画、视频、纹理和部分免署名素材。
- Wikimedia Commons：适合公共版权文物、宗教造像、古籍、历史图像，但必须逐张核对许可和署名要求。
- 博物馆开放馆藏：适合古籍、壁画、器物、宗教艺术；只使用明确标注 public domain、CC0 或允许再利用的素材。
- Openverse：适合跨站搜索 CC 授权素材；必须回到原始页面核对许可。

常见主题映射：

- 道家/《阴符经》/玄学哲思：山雾、岩壁、古籍纸纹、星图、洞窟壁面、道人背影金描线稿；避免廉价八卦、仙侠感和强符咒。
- 佛学/禅修/心性：佛像法相轮廓、莲花、圆相、经卷纹、石窟壁面、静坐身影；庄重克制，避免过度金光。
- 儒学/人文/经典阅读：古籍、竹简、碑帖、朱批、书桌光影、印章、纸纤维、儒者侧影线稿。
- 哲学/原则/思辨：石阶、门、迷宫、镜面、天平、书页、孤立坐标、极简人物线稿；避免抽象空泛。
- 心理学/心智/情绪：脑图轮廓、内在房间、镜中身影、情绪波纹、呼吸光、关系线；避免医疗图解感和鸡汤插画。
- AI/Agent/工程信号：服务器、芯片、工作台、线缆、日志屏、抽象节点图、工程空间。
- 投研/商业战略：报告纸张、交易屏、城市俯瞰、资料夹、图表纹理、会议桌局部。
- 个人品牌/创作者：工作室、笔记、相机、作品墙、桌面、人物剪影、声纹。

强度控制：

- 背景图层默认透明度 8%-22%，文化类可到 30%，但不得影响文字对比度。
- 封面和结尾可以更强，中段讲解保持更轻。
- 默认不运动或只做轻显影；慢推拉、光扫、裁切推进、线稿描边只用于封面、结尾或用户明确要高视觉版本。
- 如果主题图层比观点更显眼，必须降级为纹理、降低透明度或删掉。

素材落地记录：

每条使用外部素材的视频，必须在对应设计卡或 `docs/visual-design-cards/` 里记录：

```text
素材名称：
来源网站：
原始 URL：
作者/上传者：
许可类型：
下载日期：
用途：
处理方式：
```

实验风格规则：

- 默认风格集合只包含上面的稳定风格，不因为学习了三方 skill 自动扩展默认输出。
- 新学到的视觉方向统一沉淀到 `references/style-presets.md` 的“实验风格预设”。
- 只有用户明确要求“测试新样式”“换一种风格”或指定某个实验风格时，才从实验风格预设里选择。
- 做实验风格时必须保留旧版基准或旧关键帧，便于比较，不直接覆盖默认能力。

## 设计约束绑定

如果项目已有设计系统、色板、组件或用户认可过的版本，优先绑定它们：

- 色板绑定：不要发明随机 hex；必须从受控色板或已有组件色值里取。
- 版式绑定：延续旧视频里有效的标题位置、信息密度、字幕区域和安全边距。
- 动效绑定：延续已确认有效的运动语言，例如推进、锁定、扫描、显影。
- 资产绑定：只把实际用到的图片、音频、图标放进项目可引用目录，不依赖本机临时路径。
- 版本绑定：重大视觉改版保留旧版本或旧关键帧，便于对比，不直接覆盖所有可回退产物。

## 哲学类口播的版式节奏

儒释道、哲学、心理学、心法类视频通常不是信息表格，而是观点推进。画面应保留呼吸感，但空白必须有叙事功能。

推荐结构：

- 顶部章节锚点：编号 + 加长强调线 + 章节名。字号比普通 metadata 略大，约等于正文字号的 70%-80%；线条要足够长，成为稳定锚点，但不能抢主标题。
- 左侧观点区：主标题、短副标题、1-2 行正文摘要，正文不能小到像脚注；手机端可读性优先。
- 空区金句底纹：当左下或中下方有大面积空白时，放一条低透明大字金句，例如“能静，是修行”。透明度约 10%-18%，只做气质和节奏，不承担正文阅读。
- 关键词节点：把原来的普通小标签升级为 3-4 个时间节点，随口播逐个显影；每个节点可带 `01/02/03` 编号和轻微填充进度。
- 右侧主题图形：保留天平、星图、器物、门、经卷等隐喻图形，作为视觉解释，不和左侧正文争抢。

使用判断：

- 如果空区看起来像“缺内容”，加入金句底纹或关键词节点。
- 如果空区看起来像“安静留白”，不要硬填正文。
- 如果节点太多导致像仪表盘，删到 3-4 个关键词。
- 如果大字底纹影响可读性，先降透明度，不要直接换成装饰图。

## 多方案规则

当用户只是要求“优化风格”“更高级”“去 AI 味”时，先给 2-3 个视觉方向供选择；当用户已经明确选择了风格或说“就按这个来”时，默认只推荐一个方向并执行。

方向描述必须具体：

- 不说“更现代”，要说“更平面、更少阴影、更强网格、更少装饰线”。
- 不说“更高级”，要说“低饱和纸色底、单一强调色、标题强对比、动效只做判断显影”。
- 不说“更有动效”，要说“每段只放一个可见运动主语，关键帧能看出变化”。

## html-video 视觉参考提炼

只把这些当作视觉语言库使用。不要复制 html-video 的模板结构、文件结构、播放器、CLI 或 HTML 渲染方案。

### NYT 编辑数据图

适合：数据证据、趋势解释、研究型口播、AI 信号里需要“可信图表”的段落。

视觉 DNA：

- 底色：暖白 `#FAF9F6`、`#F7F5EE` 或深墨黑 `#0E0E0E`。
- 主文字：深灰 `#333333`，辅助文字 `#666666`、`#999999`。
- 图表线：蓝 `#326FA8`，或单一编辑强调色 `#A91D1D`、`#5FB38A`、`#D97757` 三选一。
- 字体：标题可用衬线气质，参考 `Libre Baskerville`；正文和标注用 `Libre Franklin` 或系统 sans。
- 版式：上方 kicker，小字号来源线，主体图表留大空白，注释靠近数据点。

使用规则：

- 数据图只服务一个判断，不做多图并列。
- 颜色以 ink + 1 个 accent 为主。
- 不用发光、玻璃、HUD 和装饰渐变。

### Swiss / Vignelli 网格

适合：战略框架、方法论、组织结构、步骤拆解、结论收束。

视觉 DNA：

- Swiss：灰白底 `#F2F2F2` / `#F5F5F5`，海军蓝 `#0A1E3D`，金色 `#D4A017`。
- Vignelli：白 `#FFFFFF`，黑 `#000000` / `#1A1A1A`，唯一红 `#CC0000`。
- 字体：`Helvetica Neue`、`Helvetica`、`Arial`、系统 sans；中文用系统黑体，不做 CJK uppercase。
- 版式：强网格、粗细规则线、大字号标题、编号、边栏和底栏。
- 节奏：规则线进入、标题落位、重点块锁定。

使用规则：

- 一幕只用一个红或金强调，不要红金同屏抢焦点。
- 网格是对齐系统，不是满屏线框装饰。
- 适合我们“战略作战室风”的更理性版本。

### Electric Studio 分屏

适合：高能 quote、使命陈述、强观点开头、二元对比。

视觉 DNA：

## 选色规则

- 不知道选什么：墨水经典。
- AI/技术/Agent/模型信号：靛蓝瓷或瑞士克莱因蓝。
- 战略、控盘、商业判断：墨水经典或沙丘。
- 组织、文化、长期主义：森林墨。
- 风险、验证、危机、决策窗口：瑞士安全橙或墨水经典加红色强调。
- 数据证据、趋势和研究：NYT 编辑数据图。
- 框架、方法论和结论收束：Swiss / Vignelli 网格。
- 强观点 quote：Electric Studio 分屏或 Bold Poster 欧洲海报。
- 文化纵深、东方审美、历史感：Dunhuang Mineral 敦煌矿物壁画。
- 反思、心智模型、长期主义慢叙事：Zen Negative Space 禅修留白。
- AI 信号、模型发布、未来感转场：Deep Space Signal 深空信号。
- 中文表达、人文创作、个人品牌：Ink Humanities 笔墨人文竖排。
- 审美原则、创作者反思、轻哲学：Japanese Wabi-Sabi Atlas 日式侘寂图谱。
- 中式审美总览、文化品牌开场：Chinese Dark Curation 中式暗场策展。
- 概念百科、术语拆解、系列教学：Chinese Digital Lexicon 中式数字辞典。
- 结尾关注、品牌收束、产品短尾：ReelOS Deep Space CTA 品牌深空收束。
- 一条视频只选一套主色板；允许 1 个主强调色和最多 2 个辅助强调色。
- 瑞士风只允许一个高饱和 accent，不加渐变、阴影、玻璃感和多彩混搭。

## 字体参考

优先使用系统字体保证 Remotion 渲染稳定；外部字体只作为气质参考，不默认引入网络字体。

### 中文

- 标题：系统黑体栈，字重要足，避免过细。
- 正文：系统 sans，28-38px，行高 1.25-1.45。
- 强观点海报：可用更厚的黑体或宋体气质作为副标题，但必须先看关键帧可读性。
- 中文不要强制 uppercase，不要负字距，letter-spacing 默认 0。

### 西文 / 数字 / 标签

- Swiss / Vignelli：Helvetica / Arial / system sans。
- NYT 数据：Libre Franklin 气质的 sans + Libre Baskerville 气质的 serif。
- Electric：Manrope 气质，圆润且现代。
- Tech Cursor：JetBrains Mono / system mono。
- 敦煌/中式暗场/中式辞典：Noto Serif SC / Songti SC / 系统宋体气质；正文可用系统黑体。
- 禅修/日式：Noto Serif SC/JP 气质标题 + 系统 sans 正文。
- 笔墨人文：标题可少量手写气质，正文必须 Noto Serif SC 或系统宋体。
- 深空/ReelOS CTA：宽体 display 或 mono 只用于英文状态和标签，中文标题用系统重黑体。

实践规则：

- 数字、编号、元信息可用 mono；正文不要全 mono。
- 中文标题优先稳，不追求西文字体的奇特性。
- 字体风格必须服务内容气质：数据要可信，战略要冷静，强观点要有压迫感，品牌要有留白。

## 主题节奏

视频不是每一幕都同一种底色。8 段以上建议这样安排：

```text
深色封面 → 浅色解释 → 浅色结构 → 深色转折 → 浅色展开 → 深色验证 → 浅色编排 → 深色收束
```

规则：

- 连续 3 幕以上同一明暗主题会疲劳，必须打断。
- 深色幕用于钩子、转折、风险、验证、最终金句。
- 浅色幕用于解释、拆解、流程、方法。
- 每幕只突出一个视觉动词：进入、扫描、压缩、推进、显影、拼合、验证、锁定。

## 字体与层级

- 中文标题用粗黑体系统栈，避免过细或字重飘。
- 正文控制在 28-38px，标题根据字数在 64-92px。
- 元信息、编号、ticker 用等宽字体。
- 不要在小面板里使用 hero 级字号。
- 数字、步骤、标签可用 mono，但正文不要全 mono。

## 动效设计

动效必须有可说清的作用：

- 扫描：表示发现信号、审查风险。
- 压缩：表示执行成本下降。
- 推进：表示 Agent 工作流连续执行。
- 显影：表示判断层浮现。
- 拼合：表示上下文重建现场。
- 闸门：表示验证和准入。
- 连线：表示多 Agent 编排。
- 锁定：表示最终结论沉淀。

如果关键帧看不出变化，说明动效太弱；导出中间帧重查。

## 预览验证闭环

每次完成 Remotion 组件后必须做视觉验证，优先级如下：

1. 导出封面帧，检查第一眼是否明确、有标题、有主题气质。
2. 导出至少一张动效中间态，检查运动是否真的可见。
3. 导出高信息密度帧，检查文字是否溢出、遮挡、拥挤。
4. 导出结尾帧，检查金句和收束是否干净。
5. 如有浏览器预览能力，打开 Studio 或当前页面目检。

发现问题后直接修代码，不要只在总结里提醒。

验收标准：

- 能看出这条视频属于哪种内容类型。
- 色彩不是随机拼接。
- 动效跟口播逻辑有关。
- 画面没有明显线框感、模板感、AI 蓝紫感。
- 标题、字幕、结构信息都在安全区域内。

## 禁止

- 每一幕都长得一样。
- 直接接受用户随机 hex 混搭。
- 满屏 AI 蓝紫渐变。
- 假科技 HUD 和细线框堆叠。
- 一屏多个高饱和色互相抢。
- 为了动效而动效，和口播逻辑无关。
- 只凭主观描述重做风格，不看用户已经反馈过的版本。
- 把外部设计 skill 的页面结构、素材或专有表达直接搬进视频。
