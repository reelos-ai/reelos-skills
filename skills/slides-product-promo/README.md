# Slides Product Promo

把真实网站或产品资料制作成一支“先有口播、再按声音设计画面”的产品宣传片。

这个 skill 会完成官网研究、营销口播、TTS、逐句时间轴、Bolt Slides 动效页面、MP4 渲染和音画同步检查。它适合网站宣传视频、产品发布短片、AI 产品介绍、TTS 配画面，以及需要域名输入动画的结尾 CTA。

## 核心特点

- **先研究再表达**：从官网、About、产品文档和真实 UI 中提取定位与能力，不虚构数据。
- **先生成声音再做画面**：以最终 TTS 时长为唯一时间基准，不把口播强行拉伸到固定时长。
- **按语义切镜**：产品名、功能词、转折句和 CTA 都有明确画面落点。
- **特效导演流程**：用 Motion FX Director 为每个镜头设计主特效、辅助动效、氛围层和精确触发帧。
- **Cinematic FX Mode**：选择 3–5 个 Hero Moment 制造宣传片级名场面，而不是全片平均堆特效。
- **动感但不闪眼**：减少无意义快切，在停留画面中加入推近、漂移、扫描、进度和局部循环动效。
- **可交互也可导出**：保留 Bolt Slides 演示版本，同时输出干净的 1080p MP4。
- **量化验收**：使用 100 分制评估音画同步、Hero 特效、运动工艺、品牌一致性与观看舒适度。

## 两种制作模式

| 模式 | 适合场景 | 动效策略 | 发布门槛 |
|---|---|---|---:|
| **Standard Mode** | 产品介绍、官网演示、功能讲解 | 每镜头一个主动作，强调清晰、稳定和同步 | 85/100 |
| **Cinematic FX Mode** | 产品发布、品牌宣传、需要强记忆点的短片 | 3–5 个 Hero Moment，先制作 FX Beat Map，再执行特效导演与专项验收 | 90/100 |

没有特别指定时使用 Standard Mode；用户提出“最强效果”“大片感”“发布片”或明确点名 Cinematic FX Mode 时，启用高动效流程。两种模式都以最终 TTS 为时间基准，都限制无语义运动和高频闪烁。

## 如何使用

在 Codex 中直接调用 skill，并提供要研究的网站：

```text
使用 $slides-product-promo，研究 https://reelos.ai，
先生成营销口播和 TTS，再制作一支音画同步、动感但不闪眼的产品宣传片。
```

如果你对成片已有明确要求，可以一次写完整：

```text
使用 $slides-product-promo，研究 https://reelos.ai。
制作一支 16:9、约 45–60 秒的中文产品宣传片：
- 男声，专业但有行动感
- 深色科技视觉
- 使用官网真实截图和动态素材
- 页面跟随口播语义变化
- 静止画面保持轻微运动
- 结尾加入 reelos.ai 的打字输入动画
- 输出 1080p MP4
```

也可以先让 skill 只完成前半段，确认后再继续：

```text
使用 $slides-product-promo 研究 https://reelos.ai。
先给我官网研究摘要、三版营销口播和建议音色，暂时不要制作画面。
```

确认文案后继续：

```text
采用第 1 版口播，生成 TTS；根据真实音频时间戳制作 Slides 和 MP4。
画面切换不要太密，功能介绍页加入局部持续动效。
```

要求宣传片级高动效：

```text
使用 $slides-product-promo 的 Cinematic FX Mode，研究 https://reelos.ai。
先生成口播和 TTS，再由 Motion FX Director 设计 FX Beat Map。
全片选择 3–5 个 Hero Moment，使用数据绘制、UI 叙事、镜头穿梭和域名输入等特效。
完成 MP4 后按 100 分制验收，低于 90 分继续返工。
```

## 建议提供的信息

只提供网址也可以开始。以下信息越完整，结果越贴近预期：

| 信息 | 示例 |
|---|---|
| 产品网址 | `https://reelos.ai` |
| 目标受众 | AI 创作者、独立开发者、创业团队 |
| 视频用途 | 官网首屏、社交媒体、发布会暖场 |
| 画幅 | 16:9 横屏或 9:16 竖屏 |
| 目标时长 | 30 秒、45 秒、60 秒左右 |
| 语言与音色 | 中文男声、专业、克制、有行动感 |
| 视觉方向 | 深色科技、品牌原生、编辑感、极简 |
| 必须出现的内容 | 产品名、关键功能、真实界面、域名 CTA |
| 禁止内容 | 未证实指标、竞品攻击、过度闪烁 |

## 标准工作流

```text
官网研究
  ↓
营销口播
  ↓
生成或导入 TTS
  ↓
提取真实音频时间戳
  ↓
建立逐句分镜与同步锚点
  ↓
Motion FX Director 设计特效策略
  ↓
生成逐帧 FX Beat Map
  ↓
制作 Bolt Slides 与宣传片级动效
  ↓
逐帧捕获并合成 MP4
  ↓
四轮验收与 100 分制评分
```

### 1. 官网研究

读取首页、About、产品文档和可见 UI，整理：

- 一句话定位
- 目标用户与使用场景
- 3–5 个经过验证的产品能力
- 产品工作机制或核心流程
- 品牌颜色、字体、Logo、域名和真实素材
- 可以作为事实表达的内容与只能作为营销解释的内容

### 2. 营销口播

默认采用“问题钩子 → 产品亮相 → 核心机制 → 用户收益 → 行动 CTA”的结构。中文口播以自然、可呼吸、每句话只表达一个重点为准。

### 3. TTS 与时间轴

先确认文案、声音、情绪和语速，再生成最终音频。随后从实际音频提取停顿或词句时间戳，生成 `timeline.json`。最终镜头数量和时长由声音决定。

### 4. 特效导演与 FX Beat Map

在正式制作页面之前，Motion FX Director 会完成：

- 标记产品名、功能词、转折词、数据变化和 CTA 的声音锚点
- 判断每个镜头属于 Hook、Reveal、Mechanism、Proof、Transition 或 CTA
- 为全片选择 3–5 个 Hero Moment
- 给每个镜头配置最多一个主特效、一个辅助动效和一个低注意力氛围层
- 明确特效的开始帧、冲击帧、结束帧、缓动、图层、素材要求与降级方案
- 输出 `fx-beat-map.json`

支持的主特效包括：

- 文字：逐字卡拉 OK、标题扫光、翻牌数字、Kinetic Typography
- 数据：K 线或折线绘制、雷达扫描、仪表盘、热力矩阵、SVG 流向粒子
- UI：幽灵鼠标、爆炸拆解、聚光灯、Before/After 擦除、截图飞入拼合
- 镜头：Zoom-through、Whip Pan、单帧信号闪光、Blur-through
- 氛围：胶片颗粒、透视网格、呼吸辉光、地图点位脉冲

### 5. Slides 与动效

每个镜头都需要定义：

- 对应口播
- 画面目的
- 素材来源
- 入场动作
- 停留时的持续运动
- 切换方式
- 同步锚点与允许误差

动效优先服务信息理解。产品 UI 可以使用扫描、聚焦框和缓慢推近；流程可以使用节点点亮和进度流动；域名 CTA 使用逐字符输入并保留完整域名。

### 6. MP4 渲染与质检

默认交付规格：

- 1920×1080 或 1080×1920
- 30fps
- H.264，`yuv420p`
- AAC 48kHz
- 综合响度约 -16 LUFS
- True Peak 不高于 -1.5 dBTP

导出前后会检查时间轴连续性、关键同步点、画面清晰度、音轨、黑帧和完整解码。

### 7. 验收评分

最终成片执行四轮审查：

1. 事实与营销叙事
2. 静音视觉审查
3. 音画同步审查
4. 技术交付审查

总分 100 分：

| 维度 | 分值 |
|---|---:|
| 叙事与营销清晰度 | 15 |
| 音画同步 | 20 |
| Hero 特效冲击力 | 15 |
| 运动工艺 | 15 |
| 产品证据与 UI 清晰度 | 10 |
| 品牌一致性 | 10 |
| 可读性与观看舒适度 | 10 |
| 技术交付 | 5 |

Cinematic FX Mode 必须达到 90 分，并同时满足音画同步至少 17/20、运动工艺至少 13/15、观看舒适度至少 9/10。

## 默认交付物

根据任务需要，skill 会生成以下内容：

```text
project/
├── research-notes.md       # 官网研究与事实依据
├── voiceover-brief.md      # 最终口播与声音参数
├── timeline.json           # 真实音频驱动的镜头时间轴
├── fx-beat-map.json        # 特效导演逐镜头设计
├── acceptance-report.md    # 四轮验收与量化评分
├── public/                 # 官网截图、视频、Logo、TTS 等素材
├── src/                    # Bolt Slides 页面与动效
├── scripts/                # 捕获、验证与渲染脚本
└── renders/
    └── product-promo.mp4   # 最终成片
```

## 常用调整方式

初版完成后，可以直接描述你看到的问题：

```text
画面切换稍慢，请根据口播语义增加 3–4 个镜头，但不要连续快闪。
```

```text
官网界面停留时有点静，请加入轻微推近、光标移动和局部数据变化。
```

```text
结尾域名出现得太突然，改成输入框聚焦、逐字输入、完成后停留一秒。
```

```text
官网背景视频有卡顿，请检查素材帧率、浏览器逐帧 seek 和编码参数。
```

## 运行环境

完整的 MP4 工作流通常需要：

- Codex 与 `$slides-product-promo`
- Node.js 与 npm
- Chromium 或 Chrome
- FFmpeg 与 ffprobe
- 可用的 TTS skill，或用户提供的音频

如果只需要口播、分镜或交互 Slides，可以不执行最终 MP4 渲染。

## 使用原则

- 不虚构产品能力、客户评价、增长数据或保证性结论。
- 不以低码率换取小文件，优先保证 UI 文字清晰。
- 不通过大幅改变语速强行匹配视频长度。
- 不在两秒内连续使用超过三次硬切，除非用户明确要求。
- 不修改 Bolt Slides 的 `src/deck/` 引擎目录。
- 最终交付前必须抽查关键帧和 CTA 输入动画。
