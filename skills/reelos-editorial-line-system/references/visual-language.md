# Visual Language

Use this reference for every Editorial Line System output.

## Style DNA

- Minimal black-and-white line art characters.
- Clean flat geometric proportions, simple faces, stylized bodies.
- Everyday urban lifestyle scenes: commuting, phone use, reading, shopping, selfies, walking, resting, listening, coding, deploying, presenting, reviewing, multitasking.
- Editorial design language: magazine-like typography, asymmetrical hierarchy, large negative space, strong layout blocks.
- Mostly monochrome characters. Never make the character itself colorful unless the user explicitly requests it.
- Flat vector-like output, no realistic lighting, no 3D, no glossy rendering, no busy gradients.

## Accent Palette

Use accents selectively and functionally. The default editorial palette is:

- Soft yellow: attention, approval, access, warmth, highlighted objects.
- Muted purple: infrastructure, digital systems, product surfaces, interface areas.
- Warm orange: tension, risk, warning, fracture, transition.
- Muted pink: community, human trust, creative contrast.
- Cream/off-white: background, quiet editorial whitespace.

Avoid using more than three accent colors in one image unless the user asks for a system board.

For more specific ReelOS palettes, read `reelos-style-modes.md` and choose one dominant mode. Do not mix several modes unless the user explicitly asks for comparisons.

## Composition Principles

- Let typography organize the page, but do not rely on tiny text for meaning.
- Prefer one strong visual metaphor over many weak icons.
- Use multi-panel composition when the input includes multiple clauses, scenes, stakeholders, or causal steps.
- For abstract ideas, turn each clause into an everyday action that embodies the concept.
- Characters should feel like editorial actors, not mascots.

## Negative Constraints

Avoid realistic portraits, celebrity likenesses, real company logos unless the user explicitly asks and it is appropriate, 3D renders, glossy lighting, painterly texture, anime style, childish mascot style, excessive color, cluttered dashboard tropes, floating decorative blobs, heavy gradients, and illegible typography.
