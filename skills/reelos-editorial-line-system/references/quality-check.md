# Quality Check

Before final output, check the result against this list.

## Concept

- Can a viewer understand the core matter in one glance?
- Does the image answer a central question, not merely decorate a topic?
- Is there a clear old logic, new logic, mechanism, and takeaway when the input is analytical?
- For article/signals/playbook explainers, does the image show before/after contrast, the turning point, the core mechanism, the production components, and a bottom-line judgment?
- Did the output transform the input into a visual system, not merely paste the original prompt?
- For complex analysis, is the causal chain visible?
- Is the central conflict clear in one glance?
- Does each panel or scene have a reason to exist?
- Is the conclusion visible through scene logic, not only through text?
- Would removing decorative elements leave the explanatory structure intact?

## Style

- Are characters mostly black-and-white?
- Are character forms clean, flat, geometric, and editorial rather than mascot-like?
- Are pastel colors selective and functional?
- Is there enough negative space?
- Is the layout editorial, with clear hierarchy and strong typography?

## Text

- Is visible text short, large, and readable?
- Are important Chinese titles, labels, captions, legends, and conclusion strips typeset with real text rather than drawn by image generation when sharpness matters?
- If there are more than 8 labels or any small diagram text, was the Text-safe or HTML/SVG workflow used?
- If exact text matters, was it preserved exactly?
- If long text matters, was HTML or a separate typesetting path considered?

## Safety And Specificity

- Avoid unsupported factual certainty for live news or disputed claims.
- Avoid real likenesses unless explicitly requested and appropriate.
- Avoid real logos unless the user asks and the composition needs them.
- Avoid clutter, tiny labels, decorative gradients, 3D, glossy effects, and busy backgrounds.
